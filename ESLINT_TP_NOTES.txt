ANALYSE STATIQUE AVEC ESLINT
============================

On a appris qu'ESLint est un outil d'analyse statique qui détecte les erreurs 
sans exécuter le code. Il nous aide à trouver les variables non utilisées, 
les erreurs de type et les mauvaises pratiques. C'est un outil qui s'intègre 
à l'IDE pour nous donner un retour immédiat.

CONFIGURATION DANS LE PROJET
On a vérifié qu'ESLint 9.39.2 est installé avec les plugins TypeScript et Vue.
On l'utilise pour analyser les fichiers .ts et .vue. La configuration se trouve 
dans le fichier eslint.config.js.

TEST EFFECTUÉ
On a exécuté ESLint et on a découvert une erreur : une variable 'b' non utilisée 
dans Board.vue. On a remarqué que l'IDE remonte l'erreur en temps réel. Quand on 
lance la CLI avec pnpm eslint ./app --ext .ts,.vue, ESLint confirme l'erreur.

RÉSULTAT
On a vu qu'ESLint fonctionne et détecte les erreurs avant la production
On a confirmé que l'intégration IDE fonctionne bien
On est prêt à l'utiliser dans le pipeline CI/CD
