Nous n'avons pas pu réaliser ce TP car nous n'avions pas les droits suffisants pour créer une Azure Static Web App sur notre souscription. Cependant, ce TP avait pour but d'apprendre à déployer une application Nuxt sur Azure en distinguant deux environnements (QA et production), avec un pipeline CI/CD automatisant le déploiement QA et exigeant une validation humaine avant la production. On aurait également appris à séparer le build du déploiement en réutilisant un même artifact, à stocker les deployment tokens comme secrets CI par environnement, et à utiliser l'Azure CLI pour provisionner les ressources de façon reproductible et automatisable.
